0.2 최종 확정 기준
1. 머지 보드는 6열 x 4행, 총 24칸으로 확정한다.
2. 4x4, 16칸, 기본 8칸 시작, 슬롯 확장 중심 구조는 폐기한다.
3. 하단 머지 보드는 전투 배치판이 아니라 성장 조작판이다.
4. 상단 전투 영역에는 하단 보드에서 가장 높은 등급의 물정령 1체가 직접 출전한다.
5. 동등 최고 등급 유닛이 여러 개일 경우, 먼저 생성된 유닛을 대표 출전 유닛으로 지정한다.
6. 하단 보드의 나머지 유닛은 직접 공격하지 않고 머지 재료로 남는다. 보드 전체 공격력 합산 방식은 MVP에서 사용하지 않는다.
7. 모바일 세로 화면을 기준으로 설계한다. 웹 프로토타입은 가능하지만 최종 UX 기준은 모바일이다.
8. MVP는 챕터 1, 캐릭터 5단계, 일반 적군 3종, 보스 1종으로 제한한다.

3. 짧은 플레이에서도 성장감, 보상감, 다음 단계 기대감을 제공한다.
4. MVP는 작게 시작하되 캐릭터, 보스, 챕터, 스킨, 시즌 이벤트로 확장 가능한 구조를 둔다.

2. 핵심 콘셉트 및 차별점
2.1 핵심 콘셉트
1. 하단 6x4 머지 보드에서 같은 등급의 물방울을 합쳐 상위 물정령을 만든다.
2. 하단 보드에서 가장 높은 등급의 물정령이 상단 전투 영역에 직접 출전한다.
3. 진화 단계가 올라갈수록 외형, 공격 방식, 이펙트, 데미지 표현이 강화된다.
4. 전투는 자동으로 진행되며 플레이어의 핵심 조작은 소환과 머지다.
5. 불꽃 병사 처치 보상으로 다시 소환과 강화를 반복한다.

1. 머지, 방치, 키우기류 게임에 익숙한 모바일 캐주얼 유저
2. 복잡한 컨트롤보다 성장과 수집을 선호하는 유저

4. 광고 시청 보상, 반복 강화, 도감 완성에 거부감이 낮은 유저

4. 세계관 및 스토리
4.1 기본 세계관
물의 숲은 오래전부터 작은 물방울 정령들이 지켜온 평화로운 공간이다. 어느 날 불의 군단이 숲의 수원지를 말리기 시작하고, 가장 작은 물방울 정령이 동료들과 합쳐 더 강한 물정령으로 진화하며 불의 침공을 막아낸다.
4.2 스토리 톤
1. 밝고 귀여운 캐주얼 판타지 톤을 유지한다.
2. 적은 위협적이지만 과하게 어둡거나 잔혹하게 표현하지 않는다.
3. 일반 적군은 불 모양 얼굴을 가진 갑옷 병사로 통일한다.
4. 개별 개성이 강한 불 속성 캐릭터는 일반 적이 아니라 보스로 사용한다.
5. 핵심 게임 루프
5.1 기본 루프
소환 -> 하단 6x4 머지 보드에 Lv.1 물방울 생성 -> 같은 레벨 물방울 2개 머지 -> 상위 물정령 진화 -> 최고 등급 물정령이 상단 전투 영역에 출전 -> 불꽃 병사 자동 공격 -> 적 처치 보상 획득 -> 추가 소환 또는 강화 -> 다음 스테이지 진행

6.3 전투 / 머지 메인 화면 샘플 이미지
참고: 샘플 이미지는 UI 방향성 확인용이다. 최종 구현 기준은 본 문서의 6x4 보드 및 상단 대표 출전 규칙을 따른다.
7. 6x4 머지 보드 시스템
7.1 기본 규칙
1. 보드는 6열 x 4행, 총 24개 셀로 구성한다.
2. 셀 ID는 좌측 상단부터 우측 하단 순서로 B01~B24를 사용한다.
3. 각 셀은 비어 있거나 물 캐릭터 1개를 가진다.
4. 모든 24칸은 MVP 기준 기본 사용 가능 상태로 둔다.
5. 잠김 슬롯, 슬롯 확장, 보드 확장 판매는 MVP에서 제외하고 추후 업데이트 후보로 분리한다.
6. 같은 등급 캐릭터 2개를 드래그하여 겹치거나 순차 선택하면 상위 등급 1개로 머지된다.
7. 최고 등급 캐릭터는 더 이상 머지할 수 없다.
8. 보드가 가득 차면 소환 버튼을 비활성화하고 “공간이 부족해요. 물방울을 합쳐보세요.”를 표시한다.

7.4 보드 막힘 방지 규칙
1. 머지 가능한 유닛이 있으면 해당 유닛을 항상 강조한다.
2. 빈 칸이 3칸 이하가 되면 “곧 공간이 부족해요.” 안내를 짧게 표시한다.
3. 빈 칸이 0칸이면 소환 버튼을 회색 처리한다.
4. 유닛 판매 기능은 Stage 1-5 이후 선택형으로 해금할 수 있다. 단, MVP 일정이 부족하면 판매 기능은 제외 가능하다.
5. 광고 시청 후 보드 정리, 자동 머지, 낮은 등급 일괄 정리는 출시 MVP 이후 기능으로 분리한다.

8.2 머지 처리 로직
1. 플레이어가 유닛 A를 선택하거나 드래그한다.
2. 유닛 B 위에 드롭하거나 동일 등급 유닛을 순차 선택한다.
3. 시스템은 두 유닛의 level 값을 비교한다.
4. level 값이 같고 최고 등급이 아니면 머지를 실행한다.
5. 두 유닛을 제거하고 드롭 위치 또는 선택 기준 위치에 level+1 유닛을 생성한다.
6. 새 유닛이 기존 최고 등급보다 높으면 대표 출전 캐릭터를 교체한다.
7. 머지 가능 강조, 소환 버튼 상태, 저장 데이터를 갱신한다.

9. 전투 시스템
9.1 전투 방식
1. 전투는 상단 자동 전투 영역에서 진행된다.
2. 플레이어는 전투 중 직접 유닛을 배치하지 않는다.
3. 대표 출전 물정령은 사거리 내 가장 우선순위가 높은 적을 자동 공격한다.
4. 적군은 화면 우측 또는 상단 진입로에서 등장해 방어선 방향으로 이동한다. 실제 구현에서는 한 방향으로 고정해 혼란을 줄인다.
5. 적이 방어선에 도달하면 플레이어 생명력이 감소한다.
6. 생명력이 0이 되면 스테이지 실패 처리된다.

9.3 피해 처리 순서
1. 대표 출전 유닛의 baseAttack을 가져온다.
2. 영구 강화 attackUpgradeMultiplier를 곱한다.
3. 스킬 활성화 중이면 skillMultiplier를 곱한다.
4. 적군 방어력 또는 피해 감소 수치가 있을 경우 차감한다.
5. 최종 피해량을 적 HP에서 차감한다.
6. 적 HP가 0 이하가 되면 사망 처리와 보상 지급을 실행한다.

13.2 성장 곡선
1. 초반 1~3 스테이지는 소환과 머지만으로 클리어 가능해야 한다.
2. 1-4부터는 강화 또는 Lv.3 이상 유닛 필요성을 느끼게 한다.
3. 1-7 이후에는 단순 소환 반복만으로는 부족하고 머지 타이밍이 중요해져야 한다.
4. 1-10 보스는 Lv.4 이상 대표 유닛과 1~2회 강화가 있으면 안정적으로 클리어 가능한 수준으로 설계한다.
13.3 실패 후 재도전 설계
1. 실패해도 일부 골드를 지급해 완전한 손실감을 줄인다.
2. 부족한 요소를 “공격력 부족”, “머지 단계 부족”, “보드 공간 부족”처럼 구체적으로 안내한다.
3. 광고 시청 후 1회 부활 또는 추가 보상을 제공할 수 있다.
4. 바로 재도전 버튼을 제공해 플레이 흐름을 끊지 않는다.

17. 데이터 테이블 설계 원칙
1. 모든 테이블은 한글명, 영문 필드명, 데이터 타입, 예시값, 기획 비고를 함께 작성한다.
2. 개발자가 그대로 변수명으로 사용할 수 있도록 영문 필드명은 camelCase를 사용한다.
3. 수치 조정이 필요한 값은 코드에 하드코딩하지 않고 데이터 테이블에서 관리한다.
4. MVP 데이터와 추후 업데이트 데이터를 구분하기 위해 isMvp 필드를 둘 수 있다.
5. 6x4 보드 기준으로 boardState는 24개 셀 정보를 저장한다.

19. 튜토리얼 및 초반 경험
19.1 초반 3분 목표 경험
1. 유저가 소환 버튼을 눌러 물방울을 생성한다.
2. 같은 물방울 2개를 합쳐 Lv.2 통통 물방울을 만든다.
3. 상단 대표 출전 캐릭터가 Lv.2로 바뀌는 장면을 본다.
4. 불꽃 병사를 처치하고 보상을 받는다.
5. 보상으로 추가 소환 또는 기본 강화를 진행한다.
19.2 튜토리얼 UX 원칙
1. 처음 소환과 첫 머지만 강하게 유도한다.
2. 한 번에 2줄 이상 설명하지 않는다.
3. 전투 중 긴 팝업을 띄우지 않는다.
4. 완료한 튜토리얼은 seenTutorials에 저장하고 재노출하지 않는다.

22. 스토어 문구 및 광고 소재
22.1 메인 카피 후보
1. 작은 물방울 하나로 시작하는 정화의 여정
2. 물방울을 합치면, 세상이 시원해진다
3. 불의 군단을 잠재울 가장 귀여운 물방울
4. 방치만 해도 강해지는 물방울 용사
5. 오늘도 한 방울씩 강해진다
6. 생긴 건 말랑한데, 불한테는 안 말랑함
22.2 스토어 짧은 설명
작은 물방울을 합쳐 강한 물정령으로 진화시키고, 불꽃 병사를 시원하게 제압하세요!
22.3 스토어 긴 설명 초안
작은 물방울 하나가 세상을 지킬 수 있을까요?물 키우기에서는 귀여운 물방울을 소환하고, 같은 단계의 물방울을 합쳐 더 강한 물정령으로 진화시킬 수 있습니다.진화한 물정령은 상단 전투 영역에 직접 출전해 불꽃 병사와 보스를 자동으로 공격합니다.소환하고, 합치고, 진화시키고, 보상을 모아 더 강해지세요.짧은 시간에도 성장하는 재미를 느낄 수 있는 캐주얼 머지 디펜스 게임입니다.

28. 레퍼런스 기반 기획서 수정 반영본
본 장은 아처키우기, 캣 히어로, Merge Plants, Grow Castle, Legend of Slime, Random Dice, North Tower 역기획 결과를 기존 물 키우기 기획서에 실제로 반영하기 위한 수정 기준이다.
기존 기획서의 핵심 콘셉트인 “물 캐릭터 소환 -> 6x4 머지 -> 최고 등급 직접 출전 -> 불 군단 자동 전투”는 유지한다. 단, 레퍼런스 분석 결과를 반영해 UI/UX, 성장 구조, BM, 리텐션, 데이터 테이블, MVP 우선순위를 보강한다.
28.1 레퍼런스 반영 원칙
1. 아처키우기 반영 기준
1) 머지 보드는 단순 보조 기능이 아니라 전투력 파이프라인의 핵심 성장 허브로 사용한다.
2) 소환과 머지 결과가 즉시 전투 캐릭터 외형, 공격 방식, 데미지 숫자에 반영되도록 한다.
3) 자동 제작, 자동 머지, 보스 보상, 환생형 장기 성장은 추후 업데이트 후보로 분리한다.
4) 과도한 재화, 펫, 토템, 카드, 클랜 등은 MVP에서 제외한다.
2. 캣 히어로 반영 기준
1) 전투 영역과 머지 조작 영역을 분리하되, 두 영역이 끊겨 보이지 않게 한다.
2) 하단 6x4 보드에서 플레이어가 계속 조작할 수 있게 하여 방치형이지만 손맛이 남도록 한다.
3) 동료, 룬, 스킬 조합은 장기 업데이트용으로 남기고, MVP에서는 물정령 자체 성장에 집중한다.
4) 귀여운 캐릭터성과 큰 데미지 피드백을 동시에 제공한다.
3. Merge Plants 반영 기준
1) 초반 10분 안에 유저가 설명 없이도 소환, 머지, 전투, 보상을 이해하도록 한다.
2) 신규 캐릭터 해금 시 공격력, 공격속도, 특수효과를 한 화면에 보여준다.
3) 강제 광고는 배제하고, 부스트는 보상형 광고 중심으로만 제공한다.
4. Grow Castle 반영 기준
1) 자동 전투는 낮은 스트레스와 긴 축적감을 목표로 설계한다.
2) 장기적으로는 빌드 프리셋, 던전, 비동기 랭킹을 추가할 수 있게 구조만 열어둔다.
3) 스테이지 클리어가 반복되더라도 “계속 조금씩 강해진다”는 체감을 유지한다.
5. Legend of Slime 반영 기준
1) 다층 성장 구조는 참고하되, MVP에서 시스템을 한꺼번에 열지 않는다.
2) 중복 캐릭터, 조각, 스킬 재료는 후반 자동 승급 또는 일괄 합성 기능으로 정리한다.
3) 시즌 패스, 멤버십, 던전, 유물은 추후 업데이트로 분리한다.
6. Random Dice / North Tower 반영 기준
1) 실시간 PvP, 길드 경쟁, 복잡한 장비 제작은 MVP에서 제외한다.
2) 대신 덱 조합의 재미는 캐릭터 역할 구분과 전투 스킬 차이로 약하게 반영한다.
3) 레드닷, 과도한 메뉴, 복잡한 재화 구조는 의도적으로 줄인다.

28.3 핵심 게임 구조 수정안
1. 최종 핵심 루프
소환 -> 6x4 보드에 Lv.1 물방울 생성 -> 같은 등급 2개 머지 -> 상위 물정령 생성 -> 보드 내 최고 등급 물정령 자동 출전 -> 불꽃 병사 자동 전투 -> 골드/물의 결정/조각 획득 -> 소환 레벨 또는 캐릭터 강화 -> 다음 스테이지 도전
2. 레퍼런스 반영 후 루프 강화 포인트
1) 머지 1회마다 즉시 보드 이펙트와 작은 진동을 제공한다.
2) 신규 등급 최초 생성 시 짧은 해금 팝업을 제공한다.
3) 대표 출전 물정령이 교체될 때 상단 전투 영역에서 등장 연출을 제공한다.
4) 보스전 진입 시 현재 최고 등급 물정령을 강조해 “내가 만든 캐릭터가 싸운다”는 감각을 강화한다.
5) 스테이지 실패 시 공격력 부족, 머지 단계 부족, 보드 과밀 중 하나를 안내한다.
28.4 6x4 머지 보드 수정안
1. 보드 역할 재정의
하단 6x4 보드는 전투 배치판이 아니라 성장 조작판이다. 다만 레퍼런스 분석 결과, 보드의 유닛이 전투와 완전히 분리되면 플레이어가 보드 조작의 전투 의미를 약하게 느낄 수 있다. 따라서 MVP에서는 최고 등급 1체가 직접 출전하고, 나머지 보드 유닛은 “보유 보너스” 형태로 제한적으로 전투에 영향을 준다.
2. 보유 보너스 규칙
1) 보드에 존재하는 물정령 총 레벨 합산값을 SubPower로 계산한다.
2) SubPower는 대표 출전 물정령 공격력의 최대 20%까지만 보너스로 반영한다.
3) 이렇게 하면 보드 전체가 의미를 가지되, 최고 등급 직접 출전이라는 핵심 재미가 흐려지지 않는다.
4) MVP에서 구현 부담이 크면 해당 기능은 비활성화하고 데이터 필드만 남긴다.
3. 머지 가능 강조
1) 동일 등급 2개 이상 존재 시 해당 유닛 외곽에 물결 테두리를 표시한다.
2) 유저가 3초 이상 머지하지 않으면 가장 좋은 머지 후보 1쌍을 부드럽게 점멸시킨다.
3) 5초 이상 방치 시 “같은 물방울을 합쳐보세요” 안내 문구를 하단에 노출한다.
4) 점멸 빈도는 과하지 않게 1.5초 간격으로 제한한다.
28.5 전투 시스템 수정안
1. 대표 출전 규칙
1) 보드 내 최고 등급 물정령 1체가 자동 출전한다.
2) 최고 등급이 여러 개일 경우 먼저 생성된 유닛이 대표 출전한다.
3) 같은 등급이라도 강화 수치가 높은 개체가 있다면 추후 업데이트에서 우선 출전 규칙을 확장할 수 있다.
4) 대표 출전 중인 유닛은 보드 칸에 왕관 또는 출전 아이콘을 표시한다.
2. 전투 피드백
1) 물방울 계열은 작은 물방울 투사체를 발사한다.
2) 물줄기 정령부터는 관통 궤적을 표시한다.
3) 파도 정령부터는 범위 타격 시 작은 파도 원형 이펙트를 표시한다.
4) 폭포 수호자는 타격 시 적을 0.2초 밀어내는 연출을 제공한다.
5) 얼음 물정령은 적 이동속도 감소를 파란 빙결 테두리로 표시한다.
3. 실패 원인 안내
TABLE | 실패 원인 | 판정 기준 | 안내 문구
TABLE | 공격력 부족 | 제한 시간 초과 | 공격력이 부족합니다. 물정령을 강화하세요.
TABLE | 머지 단계 부족 | 최고 등급 낮음 | 더 높은 등급의 물정령이 필요합니다.
TABLE | 보드 과밀 | 빈 칸 부족 | 보드가 가득 찼습니다. 같은 등급을 합치세요.
TABLE | 보스 대응 부족 | 보스 체력 잔여 | 감속 또는 범위형 정령을 활용하세요.
28.6 성장 시스템 수정안
1. MVP 성장축은 3개로 제한한다.
1) 물정령 강화: 대표 출전 유닛의 기본 공격력, 공격속도, 특수효과 수치를 강화한다.
2) 소환 레벨: 소환 시 등장하는 기본 물방울의 최소 레벨 또는 등장 확률을 상승시킨다.
3) 보드 효율: 소환 쿨타임, 보드 정렬, 보드 보유 보너스 효율을 강화한다.
2. 초반 강화 우선순위 설계
아처키우기 공략에서 제작 레벨과 자동 제작/자동 머지가 핵심 성장축으로 작동하는 점을 참고한다. 물키우기에서는 이를 그대로 복제하지 않고 “소환 레벨”, “소환 속도”, “머지 편의성”으로 변환한다.

TABLE | 2 | 공격력 강화 | 스테이지 진행 속도 증가 | 반영
TABLE | 3 | 소환 쿨타임 | 조작 빈도 증가 | 반영

28.7 UI/UX 수정안
1. 전투 화면 레이아웃
레퍼런스 게임들은 대부분 상단 HUD, 중앙 전투판, 하단 조작 레이어 구조를 사용한다. 물키우기도 이 구조를 유지한다. 단, 하단 메뉴가 많아지면 캣 히어로와 Legend of Slime처럼 기능 과밀이 발생하므로 MVP에서는 5개 탭 이내로 제한한다.

중앙: 대표 출전 물정령, 불꽃 병사 진입 경로, 보스 체력바, 데미지 숫자, 전투 이펙트
하단: 6x4 머지 보드, 소환 버튼, 자동 머지 버튼, 정렬 버튼, 성장/도감/상점 탭

1. MVP 리텐션
1) 일일 미션: 소환 10회, 머지 5회, 스테이지 3회 클리어, 광고 1회 시청.

1) 일일 던전: 골드 던전, 조각 던전, 보스 던전.
2) 주간 챌린지: 제한된 캐릭터 조합으로 높은 점수 도전.
3) 비동기 랭킹: 실시간 PvP 없이 점수 기반 순위만 제공.
4) 시즌 패스: 무료 트랙과 유료 트랙을 분리.
5) 컬래버레이션 스킨: 성능보다 외형과 보유 효과 중심.

28.14 개발자 전달용 변경 요구사항
1. BoardManager
1) 6x4 고정 배열을 사용한다.
2) 동일 레벨 유닛 탐색 함수를 제공한다.
3) 머지 가능 후보가 있을 때 UI에 이벤트를 전달한다.
4) 대표 출전 유닛 변경 시 BattleManager에 알린다.
2. BattleManager
1) 현재 대표 출전 유닛 데이터를 참조한다.
2) 유닛 등급별 공격 타입을 적용한다.

3. RewardManager
1) 일반 보상과 광고 2배 보상을 분리한다.
2) 오프라인 보상 누적 시간을 계산한다.

3) 해금 팝업에는 등급, 공격 타입, 특수 효과를 표시한다.
28.15 최종 판단
물키우기는 아처키우기처럼 머지 결과가 전투력으로 바로 이어져야 하고, 캣 히어로처럼 자동 전투 중에도 손이 심심하지 않아야 한다. Merge Plants처럼 초반 이해는 쉬워야 하지만 광고 의존도는 낮춰야 한다. Grow Castle처럼 장기 축적감을 주되, Legend of Slime처럼 시스템을 과하게 쌓는 것은 늦춰야 한다. Random Dice와 North Tower의 경쟁/확장 구조는 MVP 이후에만 검토한다.
따라서 v3.0 기준 물키우기의 핵심 방향은 “쉽게 시작하고, 손으로 합치고, 내가 만든 물정령이 직접 싸우며, 광고는 선택형으로만 보상하는 캐주얼 머지 디펜스”로 확정한다.
